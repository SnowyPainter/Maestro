coworker는 30초마다 스케쥴 테이블을 보고 해야할 것을 따른다.
진짜 오직 그것 뿐이다.
트렌드 데이터가 들어왔다고 해서 invoked도 아니다.

모든 것들은 사용자가 임의로 입력해서다.

트렌드 데이터를 보고 자동 드래프트, 이메일 루프도 사용자가 임의 기간동안 지정하여 스케쥴 테이블에 추가하는 식이다.
이것은 아래와 같이 표현될 수 있다.
1. 사용자가 스케쥴 테이블에 설정
2. CoWorker는 읽고 메일 보내기
3. webhook 라우터로 reply 들어오면 컴파일해서 다시 메일 전송

CoWorker 안에서 모든 루프가 이루어지는 것이 아니다!
