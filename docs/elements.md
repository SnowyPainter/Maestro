# 구성 요소 (System Components)

## 로직 엔진 (Logic Engines)

### Injector
- **역할**: 플레이북/페르소나/스니퍼 충돌 규칙에 따라 지시사항 정리 로직
- **특징**: Celery 큐가 아닌 순수 로직 처리

### LLMInterface
- **역할**: LLM 호출 인터페이스
- **원칙**: `ainvoke` 방식 사용

---

## 서비스 레이어

### Orchestrator
- **역할**: 계정 선택 여부에 따른 기능 분기 처리
- **동작 방식**:
  - **계정 선택 시**: 모든 하위 모듈 동작
  - **계정 미선택 시**: Sniffer만 동작
- **의존성**: LLMInterface를 통한 플래닝 수행
- 상태 변화가 있는 일(드래프트 생성/수정/스케줄 생성/상태 전이) 은 전부 오케스트레이터 API를 통해서만 일어나야함.

## Celery 태스크 (Background Tasks)

### Sniffer
- **역할**: 외부 트렌드 데이터 수집
- **소스**: Google Trends, Naver 등
- **특징**: 상시 작동

### Synchro
- **역할**: 개인화된 임베딩 엔진
- **범위**: 사용자별 → 계정별 메타데이터 처리

### Schedular
- **역할**: Adapter를 이용하여 스케쥴링된 작업 처리

### Generator
- **워크플로우**: `Injector` → `LLMInterface` → 지침 생성
- **역할**: 콘텐츠 생성 지침 제작

### Adapter
- **역할**: 플랫폼별 인터페이스 추상화
- **기능**:
  - 게시글 발행
  - 게시글 삭제 (가능한 경우)
  - 게시글 메트릭 수집 (모니터링)
  - Synchro로 데이터 전달
- **구조**: 플랫폼별 하위 상속 클래스들

### CoWorker
- **워크플로우**:
  1. `Sniffer` → `Generator` → 지침 이메일 발송
  2. 사용자 콘텐츠 생성 후 회신
  3. Draft 자동 저장
- **알림 기능**:
  - 발행 직전 이메일 알람
  - 사용자 선택: 무시(진행) / 취소
- **플레이북 연동**: `Synchro` → 플레이북화 → 규칙 생성 (사용자 요청 시)
