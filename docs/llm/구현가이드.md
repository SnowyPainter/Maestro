## 목표
- 사용자의 명령: "이 PostPublication에 달린 가장 뜨거운 반응의 댓글을 바탕으로 Draft 생성해줘"를 처리하는 새로운 LLM 기반 Flow 구축
- 댓글 기반 Draft 생성을 위해 LLM 서비스, 프롬프트, 어댑터, 데이터 수집 파이프라인 전반을 정비

## 선행 코드 리서치
- LLM 호출 인프라: `apps/backend/src/modules/llm/service.py` → `LLMService.instance().ainvoke(...)` 패턴, LangChain + Gemini JSON 전용, `LLMUsage` 저장 로직 포함
- 프롬프트/스키마: `apps/backend/src/modules/llm/schemas.py`, `prompt_registry.py` → `PromptKey`, `PromptVars`, Pydantic 기반 출력 스키마 검증
- 오케스트레이터 Flow 패턴: `apps/backend/src/orchestrator/flows/action/drafts.py`, `.../adapters/draft.py` → Draft 생성/업데이트 플로우, 어댑터 구현 형태 참고
- NLP 엔진: `apps/backend/src/orchestrator/nlp.py` → `IntentResult`, `FLOWS` 기반 intent 추론, 슬롯 추출 및 Flow 키와 연결 방식 이해
- 백엔드 공통 가이드: `docs/backend_development_guide.md` → LLM 사용 범위(Generator 단계), 컨텍스트 전파, 멱등성, 재시도 정책 준수 필수

## 전체 흐름 개요
1. **댓글 읽기 Flow**: 대상 `PostPublication`의 댓글/메타데이터 로드
2. **댓글 데이터 어댑팅**: LLM 입력 포맷으로 정규화
3. **LLM 호출 Flow**:
   - 프롬프트 키 선택
   - LLM 서비스 호출 및 응답 수집
4. **Draft IR 어댑팅**: LLM 응답을 `DraftIR`로 변환
5. **Draft 생성 Flow**: 기존 Draft 생성 파이프라인에 IR 주입

```
댓글 수집 → CommentAdapter → LLMFlow(prompt_key=hot_comment_draft) → DraftIRAdapter → DraftCreateFlow
```

## 구현 플랜

### 1. LLM 서비스 (`apps/backend/src/modules/llm`)
- `schemas.py`: 입력/출력 모델 정의 (이미 작성된 구조 확인)
- `service.py` 신설 또는 확장:
  - `LLMClient` 인터페이스 정의 (prompt_key, inputs, metadata)
  - 내부적으로 provider (예: OpenAI, Claude 등) 선택 가능하도록 확장성 확보
  - 재시도, 로깅, latency 측정 포함
- 기존 `orchestrator/nlp.py`와 유사한 Flow 연동 방식 검토 (slot 추출과 동일한 error handling 적용)

### 2. 프롬프트 모음집 (`docs/llm/prompts` 혹은 `apps/backend/src/modules/llm/prompts`)
- 신규 prompt_key: `hot_comment_draft`
  - **Context**: PostPublication 메타, 최고 인기 댓글, 작성자 정보, 기존 Draft 여부
  - **Instruction**: 댓글의 주요 감정/의견을 반영한 Draft 생성, tone 및 style 가이드 명시
  - **Output Format**: Markdown 기반 Draft, 필수 헤더/본문 구조 정의
- Prompt 관리 방식:
  - YAML/JSON 혹은 Python dict로 스키마화
  - 버전 관리 위해 prompt 템플릿에 `version` 명시
  - 다국어 지원 필요 시 placeholder 설계
  - 나중에 AB 테스트 가능하도록 key 기반 registry 구성

### 3. Flow 어댑터 (`apps/backend/src/orchestrator/flows/llm_*`)
- **Comment→LLM Input Adapter**
  - 입력: 댓글 텍스트, 작성자 정보, 반응 수(좋아요/공감 등), 게시물 메타
  - 출력: LLM prompt variables (`top_comment`, `engagement_stats`, `post_summary`)
  - Edge cases: 댓글 없음 → graceful fallback (빈 Draft or 에러 처리)
- **LLM Output→DraftIR Adapter**
  - 입력: LLM 응답 텍스트
  - 검증: 필수 필드(제목, 본문) 존재 여부 검사
  - 출력: `DraftIR` 구조체 (기존 Draft Flow에서 사용)
  - Validation 실패 시: error raise + 모니터링 이벤트 기록
- Flow wiring:
  - `apps/backend/src/orchestrator/flows/action/drafts.py`의 `op_create_draft` 이전 단계에 신규 Flow(예: `drafts.create_from_hot_comment`) 등록
  - `apps/backend/src/orchestrator/registry.py`에 등록하여 NLP Intent 라우팅(`NlpEngine.rules`)에 반영
  - `TaskContext`에서 `ctx.ctx`로 `LlmInvokeContext` 매핑 (request_id 등 컨텍스트 전달)

### 4. Sync Metrics 확장 (`apps/backend/src/modules/sync` 예상)
- 댓글 수집 로직에 hotness 계산 방법 추가 (예: 반응 수, 최신성 가중치)
- 댓글과 반응 데이터를 별도 테이블에 적재:
  - 필드: comment_id, post_id, author_id, reaction_count, reaction_type breakdown, is_post_author
  - 마이그레이션: Alembic 스크립트 준비
- Sync 시 작성자/댓글 작성자 동일 여부 계산 후 저장 (`is_same_author`)
- 나중에 ranking 개선 가능하도록 timestamp 및 원본 metadata 보존

### Flow 설계 세부 가이드
- 기존 `nlp.py` Slot 추출 Flow 참고:
  - `FlowContext` 재사용
  - LLM Flow 실행 이전/이후 hook 정의 (logging, tracing)
- 새로운 Flow 구성 요소:
  - `CommentHotnessService`: 가장 뜨거운 댓글 선정
  - `DraftFromCommentFlow`: 전체 orchestration
    - `fetch_comments` → `select_hot_comment` → `comment_to_llm_input_adapter` → `llm_flow.run(prompt_key="hot_comment_draft")` → `llm_output_to_draft_ir` → `draft_create_flow.run(ir)`
- 에러 처리:
  - 댓글 미존재: 사용자에게 안내 메시지 or 빈 Draft 처리
  - LLM 실패: 재시도 후 실패 시 fallback (예: 기본 템플릿 Draft)
  - Adapter 검증 실패: 모니터링 알람 + 에러 반환
- 백엔드 가이드 연계:
  - 컨텍스트 전파: `LlmInvokeContext`에 `TaskContext.request_id`, `trace_parent` 등 복사 (`docs/backend_development_guide.md:3.1`)
  - 결정론적 처리 원칙 준수: LLM 호출 외 단계는 모두 규칙 기반/멱등성 (`backend_development_guide.md:1.1`)
  - 내부 이벤트: Draft 생성 완료 시 `/internal/events/draft-ready` 등 기존 이벤트 플로우 재사용

### 테스트 전략
- **단위 테스트**
  - Comment Adapter: 다양한 댓글 세트 입력 → LLM 변수 예상 출력 비교
  - LLM Output Adapter: 정상 응답/불량 응답 케이스
  - Hotness 선정 로직: 가중치 조합 검증
- **통합 테스트**
  - Mock LLM 응답으로 Flow 실행 → DraftIR 생성 확인
  - Sync Metrics: DB 마이그레이션 후 댓글 데이터 수집/조회 확인
- **Observability**
  - LLM 호출 latency, 성공률, prompt key 별 모니터링
  - 댓글 hotness 선정 로그로 디버깅 용이성 확보
