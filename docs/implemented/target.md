# 제품 타겟

## 실제 타겟

> 지식 노동자, 1인 크리에이터, 셀프 브랜딩을 하는 모든 개인.

- 글, 영상, 코드, 콘텐츠 등 “생각”을 결과물로 내는 사람들
- 스스로의 이름으로 온라인에서 발화하는 사람들
- 매일 새로운 주제·아이디어를 탐색하고, 판단하고, 기록해야 하는 사람들

이들에게 **Maestro는 “나의 판단을 복제하는 두 번째 나”**로 작동한다. 자동화가 아니라, 나의 판단력을 저장하고 되살리는 확장기억이다.

### 타겟이 될 수 없는 경우

> 조직 중심의 브랜딩 또는 대규모 마케팅 팀

- 수십 명이 동일한 데이터셋을 다루고 KPI를 세밀하게 최적화해야 하는 환경
- 이미 CRM, BI, A/B 테스트 인프라가 완비된 팀
- 즉, ‘판단을 공유해야 하는 팀’이 아니라 ‘판단을 소유해야 하는 개인’이 타겟이다.
Maestro는 거대한 브랜드 엔진을 위한 툴이 아니다. “자기 판단을 기억하고 싶은 사람”을 위한 툴이다.

## 사용 맥락

|     🕰️ 시점    | 🌱 상황                 | ✨ Maestro가 하는 일                                 |
| :-----------: | :-------------------- | :---------------------------------------------- |
|  **콘텐츠 기획 전** | “지금 무슨 트렌드가 올라오고 있지?” | **Trends RAG + Chat Input**으로 트렌드 기반 초안을 제안한다.  |
| **콘텐츠 발행 직전** | “이 톤으로 괜찮을까?”         | **Persona Editor**가 나의 말투, 금칙어, 톤을 자동 적용한다.     |
|    **운영 중**   | “댓글 대응이 너무 많다.”       | **Reactive Rules (5-min loop)**이 나 대신 반응을 이어간다. |
|  **실시간 관제** | “Copilot이 뭘 해야할지 먼저 알려줬으면…” | **Graph RAG Copilot**이 SSE로 항상 최신 ROI/Next Action 카드를 스트림하여 아무 입력이 없어도 실행 가능한 제안을 띄운다. |
|  **성과 분석 후**  | “무엇이 효과적이었지?”         | **Playbook Dashboard**가 과거의 판단과 결과를 함께 되돌려준다.   |

## 사용하는 이유

| 문제                   | Maestro의 해석과 대응                              |
| :------------------- | :------------------------------------------- |
| **매번 다시 판단해야 하는 피로** | Maestro는 내가 내린 결정을 기억하고, 다시 같은 상황이 오면 되풀이한다. |
| **반복된 수작업**          | 한 번의 플로우 설정으로 다음부터는 자동으로 이어진다.               |
| **목소리의 불일치**         | Persona가 나의 톤과 문체를 일관되게 유지한다.                |
| **데이터의 산재**          | Playbook이 나의 모든 판단을 JSON 로그로 통합하여 맥락을 되살린다.  |
| **LLM의 불안정성**        | Maestro는 AI에게 “판단”을 맡기지 않는다. AI는 문장만 다듬는다.   |

## 사용자 여정

* 나는 매일 새로운 주제와 아이디어를 찾는다.

  * 세상이 너무 빠르게 변하고, 무엇을 써야 할지 헷갈린다.
  * 트렌드와 키워드를 직접 찾아다니는 게 점점 피로해진다.
  * **그때 Maestro를 연다.**

    * “지금 무슨 트렌드가 올라오고 있지?”
    * Maestro가 대신 탐색해주고, 나에게 맞는 아이디어를 정리해준다.
  * *혼란이 조금 정리되고, 방향이 보이기 시작한다.*

---

* 이제 무엇을 쓸지 정했지만, **어떤 톤으로 표현해야 할지 망설인다.**

  * 이건 내가 말하는 방식일까? 너무 상업적인가, 너무 건조한가.
  * **Maestro의 페르소나를 불러온다.**

    * 나의 말투, 금칙어, 분위기를 반영한 페르소나가 글을 정제한다.
  * “이 톤으로 괜찮을까?” 하는 질문에, 과거 내가 썼던 글과 반응이 함께 떠오른다.
  * *결정이 조금 더 자연스럽고 일관되게 느껴진다.*

---

* 이제 초안을 작성한다.

  * 텍스트를 쓰고, 이미지를 붙이고, 링크를 달며 흐름을 만든다.
  * **Maestro의 Draft Editor**는 내가 입력할 때마다 ‘맥락’을 저장한다.
  * *창작에만 집중할 수 있고, 나머지는 Maestro가 정리한다.*

---

* 글이 완성되면 발행을 예약한다.

  * 언제, 어디에 올릴지만 정해두면 Maestro가 대신 발행한다.
  * Threads, Instagram 등 연결된 채널에 자동으로 올려준다.
  * *나는 결과를 기다리며 다음 아이디어로 넘어간다.*

---

* 발행 후, 나는 반응을 살펴본다.

  * 사람들이 어떤 댓글을 달았는지, 어떤 주제에 반응했는지 궁금하다.
  * **Maestro의 CoWorker**가 모든 반응을 일정에 맞춰 모은다.
  * “이 톤은 참여율이 높았네.”
  * “이 시간대엔 반응이 약하네.”
  * *성과를 분석하지 않아도 배움이 남는다.*

---

* 그리고 어느 날, 예전에 썼던 주제를 다시 떠올린다.

  * “그때 그 글, 다시 써볼까?”
  * **Maestro의 기억**이 나를 대신 찾아준다.

    * 어떤 맥락에서 썼는지
    * 어떤 반응을 얻었는지
    * 어떤 페르소나를 썼는지
  * *그때의 판단이 지금의 나에게 돌아온다.*

---

* 이 루프는 매일 반복된다.

  * **탐색 → 판단 → 창작 → 발행 → 반응 → 반복**
  * Maestro는 나의 판단을 기록하고, 다시 되살려준다.
  * 나는 점점 더 효율적으로, 일관성 있게, 그리고 나답게 발화하게 된다.
  * *Maestro는 나를 대신 쓰는 것이 아니라, 나의 판단을 기억하는 두 번째 나이다.*
